# Proyecto Final — Automatización con n8n y LLMs

## 📝 Descripción del Proyecto

Este proyecto implementa un workflow automático, robusto y escalable en n8n diseñado para la **gestión y análisis inteligente de reputación de marca en redes sociales**. El sistema automatiza el ciclo completo: desde la captura de menciones (tweets) hasta la clasificación de sentimiento con IA y la ejecución de protocolos de respuesta inmediata para casos críticos.

### Problema que resuelve
Las empresas reciben miles de menciones diarias en redes sociales. Identificar manualmente las quejas urgentes es ineficiente y lento. Este flujo automatiza la detección de crisis, permitiendo que el equipo humano se enfoque en resolver problemas en lugar de clasificarlos.

### Principales Funcionalidades
1.  **Captura Centralizada:** Ingreso de tweets a través de un formulario web simulación de webhook.
2.  **Base de Datos Histórica:** Almacenamiento persistente de cada interacción en Supabase.
3.  **Clasificación de Sentimiento con IA:** Uso de Gemini 2.5 Flash para determinar si un tweet es POSITIVO o NEGATIVO.
4.  **Triaje y Lógica Condicional:** Enrutamiento inteligente del flujo basado en la emoción detectada.
5.  **Análisis Profundo de Crisis (LLM 2):** Resumen y extracción de causas raíces para quejas negativas.
6.  **Alerta Temprana Automatizada:** Notificación por correo electrónico instantánea al equipo de soporte para tweets negativos.

## 🛠️ Arquitectura y Stack Tecnológico

![Diagrama del Workflow](assets/workflow_screenshot.png)

Este workflow integra las siguientes tecnologías de vanguardia:

* **n8n:** Orquestador principal del flujo (`low-code`).
* **Google Gemini 2.5 Flash (via n8n-nodes-langchain):** Modelo de Lenguaje Grande (LLM) utilizado para clasificación de texto y síntesis de información.
* **Supabase (PostgreSQL):** Base de datos como servicio para persistir la información y el estado de cada tweet.
* **Gmail API:** Canal de notificación para alertas de crisis.
* **JavaScript (Node.js):** Lógica personalizada para transformación de datos, normalización de respuestas de la IA y manejo de errores.

## 🚀 Guía de Instalación y Configuración

Sigue estos pasos para replicar el proyecto en tu propia instancia de n8n.

### Prerrequisitos
* Una cuenta de n8n (puedes usar n8n Cloud o la versión self-hosted).
* Cuenta de Google Cloud Platform (GCP) con acceso a Gemini API y Gmail API habilitada.
* Cuenta de Supabase.

### Paso 1: Configuración de la Base de Datos (Supabase)
Crea una nueva tabla en Supabase llamada `Tweets` con la siguiente estructura (puedes usar el Editor SQL):

```sql
create table public.Tweets (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  Tweet text not null,
  Sentiment text null, -- 'POSITIVO', 'NEGATIVO', o 'NEUTRAL'
  constraint Tweets_pkey primary key (id)
);
